# 🛝 Minitopia — UI/UX Design & Digital Platform Development

> **Internship Project | Ideologiq (Advanced Ideas S.R.L.), București** | February – May 2026

---

## 📌 Project Overview

**Minitopia** is a premium digital platform for a children's indoor playground in Bucharest (Str. Veseliei Nr. 10, Sector 5), targeting families with children aged 1–12. The platform covers the full customer journey — from discovery and package comparison to online booking and post-visit reviews.

This repository documents my complete internship work at digital agency **Ideologiq**, where I was embedded in the **UI/UX Design team (Team 2)** as part of a cross-functional project involving 8 student teams from ASE Bucharest. My contributions spanned Figma prototyping, CSS design system architecture, cross-team technical handoff, and direct front-end contributions.

**Live platform:** [minitopia-playground-pro.lovable.app](https://minitopia-playground-pro.lovable.app)
**Source repository:** [github.com/lucagheorghe2005-creator/Minitopia](https://github.com/lucagheorghe2005-creator/Minitopia)

## ✨ My Direct Contributions

| Area | What I Did |
|---|---|
| 🎨 **Figma Prototyping** | Full wireframes (low → high fidelity) for Pages: Evenimente, Rezervări, Contact — desktop + mobile versions |
| 🧩 **Component Design** | Event cards, package-pricing cards, Reviews section layout, CTA buttons (pill-shaped, border-radius 9999px) |
| 🎨 **Design System** | CSS token architecture: color variables (`--color-primary-pink: #D946EF`, `--color-accent-violet: #8B5CF6`), typography scale (Inter/Geist), spacing system |
| 🌗 **Dark/Light Mode** | Proposed and documented CSS Custom Properties architecture for theme switching (`body.light {}` override system) |
| ♿ **Accessibility (WCAG AA)** | `aria-label` attributes, semantic HTML5 (`<nav>`, `<main>`, `<section>`), alt text, keyboard navigation — Lighthouse accessibility score: 84/100 |
| 🔧 **CSS Refactoring** | Migrated inline styles → external stylesheets; reduced HTML file size by ~18%; enabled dark mode compatibility; BEM convention implementation |
| 🍔 **Hamburger Menu Fix** | Resolved `display: none` / CSS transition conflict on mobile (768px breakpoint) using `visibility` + `opacity` toggle via JavaScript |
| 📋 **API Spec Documentation** | Documented reservation form fields, validation rules, and API contract for Back-end team (Team 4) |

## 🧠 What I Learned

- Full **Figma workflow** from scratch: components, auto-layout, design tokens, Dev Mode handoff — learned entirely during this internship
- Designing for **conversion**, not just aesthetics: every layout decision justified by user behavior data (80% mobile traffic, 5–8 min session windows)
- **Cross-functional communication**: negotiating CSS units (rem vs px) between design and dev, writing API spec documents, facilitating sync sessions between 3 teams
- **Technical debt management**: identifying inline CSS issues, proposing a sprint task, executing the refactor collaboratively (VSCode Live Share, pair programming)
- **WCAG 2.1 Level AA** accessibility implementation in a production environment
- **Git merge conflict resolution** under deadline pressure (48h before final demo)
